# 任务运行反馈与控制设计

## 目标

优化任务会话右侧工作区的信息层级，并补齐消息提交后的运行反馈与暂停闭环。

## 核心决策

- 会话名称进入应用顶部栏左侧；时间线与输入框继续保持阅读宽度。
- 不显示正常连接状态和空闲状态。断连、协议错误仍使用现有错误界面与输入框错误提示。
- 前台运行已开始、但当前运行段尚未产生可见内容时，在时间线末尾显示“正在思考”。
- 收到安全停止请求后显示“正在暂停”。安全点完成后显示“继续”操作。
- “暂停”映射 daemon 的 `stop_run: safe_point`。它在原子操作边界结束当前运行段并保存检查点，不承诺冻结进程。
- 运行期间保留排队消息能力；暂停按钮与排队发送按钮并列。
- 强制停止不进入主操作，避免把不可回滚的副作用包装成普通暂停。

## 状态模型

| 运行状态 | 时间线反馈 | 输入区控制 |
|---|---|---|
| `running` 且当前段尚无可见输出 | 正在思考 | 暂停、排队消息 |
| `running` 且已有文本或工具活动 | 使用现有流式内容或活动 | 暂停、排队消息 |
| `waiting_permission` | 使用现有权限请求 | 暂停、排队消息 |
| `yielding` | 正在暂停 | 禁用暂停、保留排队消息 |
| `interrupted + cancelled` | 无持续动画 | 继续、发送新消息 |
| 其他终态 | 无运行反馈 | 发送新消息 |

## 布局

- 应用顶部栏在任务路由显示当前会话名称，右侧保留命令面板按钮。
- 任务工作区内部不再重复渲染标题、连接状态和任务状态。
- 阅读列从时间线开始，右侧环境摘要与阅读列共享工作区首行。

## 无障碍

- “正在思考”和“正在暂停”使用 `role=status` 与礼貌播报。
- 等待动画在减少动态效果时停止旋转。
- 暂停和继续按钮同时提供图标、可见文字和可访问名称。
- 会话名称继续作为页面 `h1`，并在主内容前出现在 DOM 阅读顺序中。

## 非目标

- 不修改 daemon 的运行控制协议。
- 不增加强制停止主按钮或撤销副作用能力。
- 不改变排队消息、权限请求和工作台对象交互。
