# 任务文件活动与 Diff 实施计划

## 1. 文件变更快照

- 为文件编辑和写入生成有行号范围的统一 diff。
- 返回 `diff`、`additions`、`deletions` 与原有结构化结果。
- 覆盖替换、创建、覆盖和无变化场景。

## 2. 时间线投影

- journal 投影保留读取文本与编辑 diff，并生成 `+N -N` 编辑摘要。
- 前端实时投影使用相同提取规则，避免刷新前后展示不一致。
- 保持命令输出和其他工具活动行为不变。

## 3. 对话内文件活动

- 文件读取与编辑活动默认展开。
- 读取活动打开文件预览；编辑活动打开完整 diff。
- 在活动组中增加可折叠的内联 diff，复用现有解析和行级语义。
- 增加中英文文案和窄屏滚动约束。

## 4. 验证

- 补充文件工具、journal 投影、实时投影、工作台目标、时间线和 diff 渲染测试。
- 增加 Storybook 文件活动场景。
- 运行 `pnpm check:frontend:fast`、相关 Rust 测试和 `pnpm check:quick`。
- 在桌面宽屏与窄屏状态下截图，并完成设计 QA。
