# 任务运行反馈与控制实施计划

## 1. 顶部会话名称

- 在应用顶部栏读取当前任务标题。
- 移除任务阅读列内的标题、连接状态和任务状态。
- 覆盖任务路由、无任务和标题更新场景。

## 2. 运行反馈

- 根据当前运行段和时间线内容派生 `thinking / pausing / none`。
- 将反馈作为时间线末端区块参与排序、虚拟化和自动跟随。
- 增加中英文文案与无障碍状态播报。

## 3. 暂停与继续

- 在任务输入组件中发送 `stop_run` 的 `safe_point` 模式。
- 对安全暂停产生的 `cancelled` 运行提供 `continue_task`。
- 保留运行时排队输入，增加暂停中和继续中防重复状态。
- 保持命令元数据幂等和流版本更新。

## 4. 验证

- 补充 AppShell、TaskWorkspace、TaskTimeline、TaskComposer 和 Composer 测试。
- 增加 Storybook 运行反馈场景并进行桌面视觉检查。
- 运行 `pnpm check:frontend:fast` 和 `pnpm check:quick`。

## 实施结果

- 会话名称已进入应用顶部栏；任务阅读列不再重复标题、连接状态和普通任务状态。
- 当前运行段尚无输出时显示“正在思考”；安全停止阶段显示“正在暂停”。
- 运行态提供安全暂停；安全暂停结束后提供继续，并沿用任务命令的幂等元数据与流版本处理。
- `pnpm check:frontend:fast` 通过：66 个测试文件、680 项测试。
- `pnpm check:quick` 已执行。任务架构超限项已拆分；检查仍被仓库内其他既有超限测试文件阻断。
