# 任务文件活动与 Diff 设计

## 目标

让用户不离开任务会话，就能看懂代理读取了哪些文件、改了哪些文件，并检查对应文件内容与 diff。

## 核心决策

- 文件读取、搜索和编辑继续属于运行段的工具活动，不新增独立消息类型。
- 文件活动组默认展开。每次调用保留紧凑单行摘要，文件名承担打开入口。
- 读取完成后可在任务工作台查看本次读取到的文本快照。
- 编辑完成后在活动组内显示“已编辑的文件”和统一 diff 预览；点击活动行可在任务工作台查看完整 diff。
- Diff 使用工具执行时生成的前后快照，不在前端读取当前工作区重新计算，避免历史记录随文件后续变化而漂移。
- 读取与 diff 预览都复用现有工作台、标签页、焦点恢复和响应式全屏行为。

## 数据链路

1. `FileRead` 完成后，时间线投影保留受长度限制的文本结果。
2. `FileEdit` 和 `FileWrite` 在写入时根据前后内容生成统一 diff，并返回增删统计。
3. journal 投影与前端实时投影使用同一规则，把读取文本或 diff 放入工具活动的 `output`。
4. 时间线把可查看的文件活动映射为 preview-only 工作台目标。
5. 工作台复用文本产物渲染器和 `DiffPanel`，不直接访问工作区文件系统。

## 交互

- 进行中的读取或编辑显示现有加载状态，不提供空内容入口。
- 完成的读取行可打开“文件”标签。
- 完成且存在 diff 的编辑行可打开“Diff”标签。
- 内联 diff 默认展开，支持折叠；长内容在固定高度内滚动。
- 没有实际替换或前后内容一致时不显示空 diff。
- 拒绝和失败的编辑只显示失败状态，不把未执行的改动展示为已完成 diff。

## 无障碍

- 文件活动入口使用完整可访问名称，例如“打开文件 src/a.rs”。
- Diff 表格保留新旧行号、增加、删除和上下文的语义。
- 折叠区使用原生 `details/summary`，键盘可操作。
- 颜色不是唯一差异信号；增删行同时保留 `+/-` 文本和屏幕阅读器标签。

## 非目标

- 不实现通用代码编辑器、语法服务器或工作区文件浏览器。
- 不从前端实时读取任意路径。
- 不为历史上没有保留输出的工具调用补造文件内容或 diff。
- 不改变工具授权与 sandbox 边界。
