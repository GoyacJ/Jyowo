# 输入框快捷面板设计

## 目标

重构输入框内的 `/` 指令与 `@` 引用选择。候选面板不再撑高输入框，并在同一套视觉和键盘模型下提供筛选、分组、说明与状态反馈。

## 视觉依据

- `/` 面板采用参考图中的指令行结构：图标、命令、名称和用途说明。
- `@` 面板采用参考图中的分组列表结构：类型图标、名称和辅助信息。
- 面板位于输入框上方，宽度与输入框一致，保持输入区常驻尺寸不变。
- 沿用 Jyowo 的 Inter 字体、Lucide 图标、语义颜色、圆角和阴影令牌。

## 信息架构

- `/` 只承载提示指令：计划、审查、修复、解释。
- `@` 只承载上下文引用：文件、产物、对话、记忆、技能、工具和 MCP 服务。
- 附件继续由回形针入口承担；权限、记忆模式和模型继续留在工具栏。
- 项目、执行环境和分支上下文不并入快捷面板，避免与任务工作区已有信息重复。

## 交互

- 在光标所在词元输入 `/` 或 `@` 时打开面板。
- 输入 `/pl` 或 `@comp` 时保持面板打开并过滤结果。
- 编辑器触发时使用编辑器内的词元作为搜索条件，不显示第二个搜索框。
- 工具栏 `@` 按钮打开面板时显示搜索框并自动聚焦。
- 方向键移动选中项；Enter 或 Tab 插入；Esc、点击外部或删除触发词关闭。
- 选择引用后显示现有引用标签，并将焦点还给编辑器。

## 布局

- 面板绝对定位在输入框上方，间距 8px，左右边缘对齐输入框。
- 最大高度 360px；候选区域独立滚动。
- 每行最小高度 44px，整行使用选中背景。
- 面板底部显示键盘提示。
- 窄屏隐藏次要说明，保留图标、主名称和选中状态。

## 无障碍

- 编辑器触发时，`aria-expanded`、`aria-controls` 和 `aria-activedescendant` 绑定到实际获得焦点的编辑器。
- 工具栏触发时，相同属性绑定到搜索输入框。
- 候选使用 listbox/option 语义；加载、空结果和结果数量使用状态播报。
- 所有可见文字进入中英文 i18next 资源。

## 非目标

- 不改变引用候选协议或 daemon 数据结构。
- 不增加新的斜杠指令。
- 不重做输入框工具栏和任务工作区上下文栏。
