# 输入框快捷面板实施计划

## 1. 统一面板结构

- 新增输入框候选面板壳层，统一宽度、定位、滚动区、选中态和键盘提示。
- 将 `/` 和 `@` 面板移出输入框内容流，固定在输入框上方。
- 工具栏 `@` 按钮复用同一个引用面板实例。

## 2. 指令筛选

- 从光标位置解析斜杠词元和查询文本。
- 按命令、中文名称和说明过滤。
- 为四个现有指令补充 Lucide 图标和中英文说明。
- 支持方向键、Enter、Tab、Esc 和光标位置替换。

## 3. 引用筛选

- 从光标位置解析 `@` 词元。
- 编辑器触发时移除重复搜索输入；工具栏触发时保留并聚焦搜索输入。
- 为七类引用补充分组本地化、类型图标和可用辅助信息。
- 选择后保持现有引用去重与标签行为，并恢复编辑器焦点。

## 4. 验证

- 补充触发词筛选、光标位置替换、键盘选择、工具栏搜索与关闭行为测试。
- 增加 Storybook 快捷面板状态，检查浅色、深色和窄屏布局。
- 运行定向测试、`pnpm check:frontend:fast` 和 `pnpm check:quick`。
- 使用浏览器捕获实现状态，与参考图进行设计 QA，并把结果追加到项目根目录 `design-qa.md`。

## 实施结果

- `/` 和 `@` 已统一使用输入框上方的单一候选面板。
- `/` 支持查询过滤、方向键、Enter、Tab、Esc 和光标位置替换。
- 编辑器内 `@` 不再显示重复搜索框；工具栏 `@` 保留自动聚焦搜索。
- 引用按项目文件、产物、对话、记忆、技能、工具和 MCP 服务分组。
- 定向测试 55 项通过，TypeScript 类型检查通过，设计变量检查通过。
- `pnpm check:quick` 已执行，被仓库内既有超限测试文件阻断；本次新增测试文件不在超限列表中。
